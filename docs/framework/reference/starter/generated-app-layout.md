# Generated App Layout

> Structure and conventions of an application created by `gofra new`.

## Status

Alpha — the generated layout may change before v1.

## Overview

Running `gofra new <directory>` produces a complete, runnable Go application.
This page documents the directory structure, key files, and runtime behavior
of that generated application.

## Directory Structure

```
<app>/
├── .dockerignore             # Build context exclusions for docker build
├── .env.example              # Optional local overrides for compose + DB tasks
├── .github/
│   └── workflows/
│       └── ci.yml            # Test + build + docker image on PR and main
├── .gitignore
├── cmd/
│   └── app/
│       └── main.go           # Application entrypoint
├── compose.yaml              # Local PostgreSQL for development
├── config/                   # Generated after running mise run generate
│   ├── config_gen.go         # Config structs and DefaultConfig()
│   ├── load_gen.go           # NewFlagSet() and Load()
│   └── public_gen.go         # BindPublicConfig() and PublicConfigHandler()
├── db/
│   ├── embed.go              # Embeds migrations into the binary
│   ├── migrations/           # goose SQL migration files
│   │   └── 00001_create_posts.sql
│   ├── queries/              # sqlc query files
│   │   └── posts.sql
│   ├── seeds/                # Seed data (goose --no-versioning)
│   │   └── seed.sql
│   └── sqlc/                 # Generated by sqlc (do not edit)
├── proto/
│   └── <package>/
│       └── config/
│           └── v1/
│               └── config.proto  # Configuration schema (protobuf)
├── scripts/
│   ├── compose.sh            # Picks Docker Compose or Podman Compose
│   ├── load-env.sh           # Loads optional .env and derives DB + ZITADEL env vars
│   ├── wait-for-postgres.sh  # Waits for Postgres readiness after compose up
│   └── wait-for-zitadel.sh   # Waits for ZITADEL /debug/healthz to return 200
├── web/
│   ├── .gitignore            # node_modules, dist (except .gitkeep), src/gen
│   ├── dist/                 # Vite build output (only .gitkeep committed)
│   ├── embed.go              # //go:build !dev — embeds web/dist
│   ├── embed_dev.go          # //go:build dev — reverse proxy to Vite
│   ├── index.html            # SPA entry HTML
│   ├── package.json          # Frontend npm dependencies
│   ├── src/
│   │   ├── components/ui/    # shadcn/ui components (copied)
│   │   ├── gen/              # Generated TS runtime config
│   │   ├── lib/              # cn() helper + runtime-config re-export
│   │   ├── main.tsx          # React entry (TanStack Query + Router)
│   │   ├── routes/           # TanStack Router code-based routes
│   │   └── styles.css        # Tailwind v4 entry with theme tokens
│   ├── tsconfig.json
│   └── vite.config.ts
├── Dockerfile                # Multi-stage build → distroless static image
├── go.mod                    # Go module manifest
├── gofra.yaml                # Default runtime configuration
├── mise.toml                 # Task runner definitions
├── sqlc.yaml                 # sqlc code generation configuration
└── README.md
```

The `config/` directory does not exist immediately after `gofra new`. It is
generated by `mise run generate` (which is a dependency of `mise run dev`,
so it runs automatically on first `dev`). The `db/sqlc/` directory is
generated by `mise run gen:sql` (requires sqlc to be installed).

## Key Files

### `cmd/app/main.go`

The application entrypoint. It:

1. Loads configuration via `config.Load(os.Args[1:])` (the generated
   wrapper, not `runtimeconfig.Load` directly).
2. Opens a PostgreSQL connection pool via `runtimedatabase.Open()`, mapping
   `cfg.Database` fields into `runtimedatabase.Config`. Duration fields
   (e.g., `max_conn_lifetime`) are parsed from Go duration strings.
3. Optionally runs embedded migrations via `runtimedatabase.Migrate()` when
   `cfg.Database.AutoMigrate` is true. Applied migrations are logged.
4. Creates a health checker with `runtimehealth.New()` including a Postgres
   readiness check via `runtimedatabase.HealthCheck(pool)`.
5. Sets up two handler layers:
   - **Root mux** (`http.NewServeMux`) — health probe handlers, structurally
     outside app middleware.
   - **App router** (`chi.NewRouter`) — config endpoint and web assets,
     mounted at `/` on the root mux.
6. Starts the server with `runtimeserve.Serve()`. The database pool is closed
   in the `OnShutdown` callback during graceful shutdown.

### `proto/<package>/config/v1/config.proto`

Protobuf definition of the application's configuration schema. Defines five
message types:

**`Config`** (root):

| Field | Type | Description |
|-------|------|-------------|
| `app` | `AppConfig` | Server-side settings |
| `public` | `PublicConfig` | Browser-safe settings served to the frontend |
| `database` | `DatabaseConfig` | PostgreSQL connection and pool settings |
| `auth` | `AuthConfig` | Server-side JWT validation settings (opt-in) |

**`AuthConfig`**:

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `issuer` | `string` | `""` | OIDC issuer URL used for JWKS fetching; empty disables auth |
| `audience` | `string` | `""` | Expected JWT audience claim; empty disables auth |

When either field is empty the server logs `auth disabled` and does not
install the JWT middleware. ZITADEL ships in the default `compose.yaml`, but
the Go binary itself does not require any ZITADEL configuration — a fresh
starter runs without touching identity settings.

**`DatabaseConfig`**:

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `dsn` | `string` | `postgres://...localhost:5432/<app>` | Postgres connection string (secret — not exposed via flags) |
| `max_conns` | `int32` | `0` (pgxpool default) | Maximum pool connections |
| `min_conns` | `int32` | `0` (pgxpool default) | Minimum idle connections |
| `max_conn_lifetime` | `string` | `""` (pgxpool default: 1h) | Go duration string for max connection reuse time |
| `max_conn_idle_time` | `string` | `""` (pgxpool default: 30m) | Go duration string for max idle time |
| `health_check_period` | `string` | `""` (pgxpool default: 1m) | Go duration string for health check interval |
| `auto_migrate` | `bool` | `false` | Run goose migrations on startup |

Duration fields use Go duration string format (e.g., `1h`, `30m`, `1m`).
Empty strings leave the pgxpool defaults in place. The `dsn` field is marked
as secret and is not exposed as a CLI flag.

**`AppConfig`**:

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `name` | `string` | App directory name | Application name for logs |
| `port` | `int32` | `3000` | HTTP server port |

**`PublicConfig`**:

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `app_name` | `string` | App directory name | Display name in the browser |
| `api_base_url` | `string` | `"http://localhost:3000"` | Base URL for API calls |
| `auth` | `PublicAuthConfig` | — | OIDC authentication settings |

**`PublicAuthConfig`**:

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `issuer` | `string` | `"http://localhost:9000"` | OIDC issuer URL |
| `client_id` | `string` | `"<app>-web"` | OIDC browser client ID |
| `scopes` | `repeated string` | `["openid", "profile", "email"]` | OIDC scopes |
| `redirect_path` | `string` | `"/auth/callback"` | Auth callback path |
| `post_logout_redirect_path` | `string` | `"/"` | Post-logout redirect |

All defaults use `gofra.config.v1.field` annotations. Run
`gofra generate config` after editing to regenerate the Go code.

### `gofra.yaml`

Default configuration overrides. Proto defaults apply when a key is omitted.

```yaml
app:
  name: <app-name>
  port: 3000

database:
  dsn: "postgres://postgres:postgres@localhost:5432/<app-name>?sslmode=disable"
  auto_migrate: true
```

This file is read by `runtimeconfig.Load` as the YAML layer. It can be
overridden by environment variables or CLI flags.

### `compose.yaml`

The local infrastructure definition for the generated app. It contains two
services:

**`postgres`**:

- pinned official image tag (`postgres:18.3-alpine3.23`)
- named volume (`postgres_data`) so `infra:stop` keeps data
- `pg_isready` healthcheck so the `infra` task can wait for readiness
- environment-variable defaults that line up with `gofra.yaml`

**`zitadel`**:

- pinned image `ghcr.io/zitadel/zitadel:stable`
- runs `start-from-init --masterkeyFromEnv --tlsMode disabled`, depending on
  Postgres being healthy
- bootstraps its own `zitadel` database and user on first start, using the
  Postgres admin credentials wired in via `ZITADEL_DATABASE_POSTGRES_ADMIN_*`
- exposed on `:8081` so it does not collide with other services that may
  claim `:8080` later (Restate, Vite preview, etc.)

The compose file uses the canonical `compose.yaml` name and is designed to work
with either Docker Compose or Podman Compose via `scripts/compose.sh`.

### `mise.toml`

Task runner definitions:

| Task | Depends On | Description |
|------|-----------|-------------|
| `generate` | — | Generates config code from proto via `gofra generate config` |
| `gen:sql` | — | Generates Go code from SQL queries via `sqlc generate` |
| `infra` | — | Starts local PostgreSQL + ZITADEL via Compose and waits until both are ready |
| `infra:stop` | — | Stops local infrastructure containers |
| `infra:reset` | — | Stops local infrastructure and removes volumes |
| `infra:logs` | — | Tails Compose service logs |
| `web:install` | — | Runs `npm install` in `web/` |
| `web:dev` | `generate` | Runs `npm install && npm run dev` in `web/` (Vite at `:5173`) |
| `web:build` | `generate` | Runs `npm install && npm run build` in `web/` (produces `web/dist/`) |
| `dev` | `generate` | Generates code then runs `go run ./cmd/app` |
| `migrate` | — | Runs pending database migrations via goose |
| `migrate:create` | — | Creates a new migration file |
| `migrate:down` | — | Rolls back the last migration |
| `migrate:status` | — | Shows migration status |
| `seed` | — | Seeds the database with development data |

The `generate` task uses `GOFLAGS=-mod=mod` so `go run` can bootstrap
`go.sum` on first invocation before `go mod tidy` finalises it. The
`migrate`, `migrate:*`, and `seed` tasks source `scripts/load-env.sh`, which
loads `.env` if present and derives `DATABASE_URL` and `GOFRA_DATABASE__DSN`
from the shared local DB defaults when they are not set explicitly.

### `.env.example`

Optional template for local overrides. It defines the Postgres image tag,
host/port, credentials, database name, and SSL mode used by:

- `compose.yaml`
- the goose-based migration and seed tasks
- `mise run dev` via `GOFRA_DATABASE__DSN`

No `.env` file is required for the out-of-the-box setup. The defaults already
match the generated `gofra.yaml`.

### `scripts/`

- **`compose.sh`** — Detects `docker compose`, `podman compose`,
  `docker-compose`, or `podman-compose` and always runs against the app's
  root `compose.yaml`.
- **`load-env.sh`** — Loads optional `.env` overrides and exports the
  database URL contract plus the `GOFRA_ZITADEL_*` variables so Compose,
  goose, and the Go app see the same values.
- **`wait-for-postgres.sh`** — Polls `pg_isready` through Compose until the
  Postgres service accepts connections.
- **`wait-for-zitadel.sh`** — Polls `GET /debug/healthz` on the host side
  (curl or wget) until ZITADEL returns 200.

### `db/`

- **`embed.go`** — Uses `//go:embed migrations/*.sql` to bundle migration
  files into the binary. Exports `var Migrations embed.FS` for use with
  `runtimedatabase.Migrate`.
- **`migrations/`** — goose SQL migration files with `-- +goose Up` /
  `-- +goose Down` annotations. File names are zero-padded
  (`00001_create_posts.sql`) so sqlc parses them in order.
- **`queries/`** — sqlc query files. Each `.sql` file contains named queries
  (`-- name: GetPost :one`) that sqlc compiles into type-safe Go code.
- **`seeds/`** — Seed data applied via `goose --no-versioning`. Written to
  be idempotent (`ON CONFLICT DO NOTHING`).
- **`sqlc/`** — Generated by `mise run gen:sql`. Do not edit.

### `sqlc.yaml`

sqlc configuration. Points `schema` at `db/migrations/` (sqlc reads goose
migration files as the database schema), `queries` at `db/queries/`, and
generates Go code into `db/sqlc/` using `pgx/v5`.

### `web/`

Full Vite + React 19 + TanStack (Router/Query/Form) + Tailwind CSS v4 +
shadcn/ui scaffold. No lint/format tooling by default — add Biome,
Prettier, or ESLint only if the app wants them.

- **`index.html`** — SPA entry. Loads `/_gofra/config.js` before
  `src/main.tsx` so `window.__GOFRA_CONFIG__` is populated before React
  mounts.
- **`package.json`** — `vite`, `@vitejs/plugin-react`, `tailwindcss@^4`,
  `@tailwindcss/vite`, `@tanstack/react-router`, `@tanstack/react-query`,
  `@tanstack/react-form`, `@connectrpc/connect`, `@connectrpc/connect-web`,
  `@radix-ui/react-slot`, `class-variance-authority`, `clsx`,
  `tailwind-merge`, `lucide-react`.
- **`vite.config.ts`** — React + Tailwind v4 plugins; `@/*` alias to
  `src/*`; builds to `dist/`.
- **`src/main.tsx`** — Wires `QueryClientProvider` and `RouterProvider`.
- **`src/routes/`** — Code-based TanStack Router tree (`__root.tsx`,
  `index.tsx`).
- **`src/components/ui/button.tsx`** — shadcn/ui `Button` copied in
  directly. Not a dependency — regular source that the app can edit.
- **`src/lib/utils.ts`** — `cn(...)` helper (shadcn convention).
- **`src/lib/runtime-config.ts`** — Re-exports `runtimeConfig`,
  `loadRuntimeConfig`, and `RuntimeConfig` from
  `src/gen/runtime-config.ts`.
- **`src/styles.css`** — `@import "tailwindcss";` plus a `@theme { ... }`
  block with shadcn-compatible OKLCH tokens.
- **`src/gen/`** — Generated by `mise run generate`. Git-ignored except
  for `.gitkeep`.
- **`dist/`** — Vite build output. Git-ignored except for `.gitkeep` so
  `//go:embed all:dist` in `embed.go` compiles before the frontend has
  been built.
- **`embed.go`** (`//go:build !dev`) — `//go:embed all:dist`, SPA-fallback
  handler that serves file matches directly and falls back to
  `dist/index.html` for client-side routes.
- **`embed_dev.go`** (`//go:build dev`) — Reverse proxy to
  `http://localhost:5173` so the Go binary can delegate to the Vite dev
  server during development.

### Deployment artifacts

Three files ship in every generated app so the code can be tested, built as a
binary, built as a container image, and exercised in CI from day one:

- **`Dockerfile`** — Multi-stage build. A `golang:1.25-alpine` builder stage
  produces a static binary with CGO disabled; the runtime stage is
  `gcr.io/distroless/static-debian12:nonroot`. The resulting image has no
  shell and runs as a non-root user.
- **`.dockerignore`** — Excludes the `.git` directory, `.env` files, editor
  state, and pre-built binaries from the Docker build context.
- **`.github/workflows/ci.yml`** — Runs `mise run test`, `mise run build`,
  and a `docker build` step (via `docker/build-push-action@v6` with
  `push: false`) on every pull request and every push to `main`.

See [Deployment](deployment.md) for the full definitions of each artifact.

### `go.mod`

The generated `go.mod` depends on a published release of the framework:

```
require github.com/Gabrielbdd/gofra v0.1.0
```

There is no `replace` directive. Generated apps resolve the framework through
the Go module proxy like any other dependency.

## Runtime Behavior

On startup, the app connects to PostgreSQL using `database.dsn`, optionally
runs embedded migrations when `database.auto_migrate` is true, and registers
a Postgres readiness check. If the database is unreachable or migrations
fail, the app exits immediately (fail-fast).

The generated app listens on the port configured in `gofra.yaml` (field
`app.port`, default `3000`). It exposes:

| Path | Handler | Source | Purpose |
|------|---------|--------|---------|
| `/startupz` | Root mux | `runtimehealth` | Startup probe — returns 200 after init |
| `/livez` | Root mux | `runtimehealth` | Liveness probe — always returns 200 |
| `/readyz` | Root mux | `runtimehealth` | Readiness probe — 200 if checks pass |
| `/_gofra/config.js` | App router | Generated `PublicConfigHandler` | Public config as JavaScript |
| `/*` | App router | `web.Handler()` | Embedded SPA assets |

Health probes are on the root mux, structurally outside the app router's
middleware chain. This ensures health checks work even if app middleware
fails.

The config handler serves the `PublicConfig` subtree only — `AppConfig`
fields (like `port`) and any secret fields are never sent to the browser.

Graceful shutdown is handled by `runtimeserve.Serve()` with the health
checker wired as the `Health` provider. See
[runtime/serve](../runtime/serve.md) for shutdown sequence details.
